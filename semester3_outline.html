Day 1
Assembly Concepts ‚Äî Constraints, Mates & the Assembly File
Intro
Students learn what an Assembly (.iam) file is and how it differs from a Part (.ipt) file. Teacher introduces assembly constraints (Mate, Flush, Insert, Angle) using a pre-built sample assembly of simple blocks. Key concept: degrees of freedom ‚Äî how each constraint removes motion from a part. Students practice adding and removing constraints on the sample file to observe how parts move.
Indiana Standards
CTE-TE 6.1 CTE-TE 5.2
CTE-TE 6.1: Digital tools to model solutions ‚Äî assemblies represent multi-part systems. CTE-TE 5.2: Interpreting a multi-part model mirrors reading a technical assembly drawing.
Vocabulary
assembly mate constraint degrees of freedom grounded part Bill of Materials
Day 2
Building a Simple Hinge Assembly ‚Äî Two Parts, One Joint
Skill
Students model two provided simple parts (a pin and a clevis bracket) and assemble them using Mate and Insert constraints. They animate the assembly's rotation to verify it behaves correctly. Teacher introduces the Bill of Materials (BOM) panel ‚Äî students see how CAD systems track parts automatically. Notebook entry: sketch the assembly and label each constraint type used.
Indiana Standards
CTE-TE 6.1 CTE-TE 3.2 IAS-Sci 8.PS
IAS-Sci 8.PS (Physical Science): Forces and motion ‚Äî a pinned joint has one rotational degree of freedom; students connect assembly behavior to physics concepts of motion and force transmission.
Assessment (Informal)
Assembly moves correctly
Constraints fully applied
Notebook sketch labeled
Day 3
Assembly Drawing ‚Äî Exploded View & Parts List
Skill
Students generate an exploded view of their hinge assembly using Inventor's Presentation (.ipn) environment, then create a Drawing file showing the exploded view with balloon callouts and a Parts List (BOM table). This is the standard format for assembly instructions used in manufacturing. Students compare their drawing to an IKEA-style instruction sheet to see real-world parallels.
Indiana Standards
CTE-TE 5.1 CTE-TE 5.2 CTE-TE 4.2
An exploded assembly drawing is a professional communication document (CTE-TE 4.2) that must follow drawing standards (CTE-TE 5.1) and represent the model accurately (CTE-TE 5.2).
Assessment (Informal)
Exploded view readable
BOM table present
Balloon callouts correct
Day 4
Mini-Assembly Project ‚Äî Design & Assemble a 3-Part Device
Project
Students design and model a simple 3-part assembly of their choice from a provided prompt list (examples: a simple clamp, a spring-loaded latch concept, a swivel base). All parts are modeled from scratch, assembled with full constraints, and documented with an exploded drawing + BOM. Students add this to their portfolio. This is the first time students manage a multi-part CAD project independently.
Indiana Standards
CTE-TE 3.1 CTE-TE 5.1 CTE-TE 5.2 CTE-TE 6.1 IAS-Math 8.G
IAS-Math 8.G: Understand congruence, similarity, and geometric transformations. Assembly constraints are spatial transformations ‚Äî rotating and mating parts connects directly to 8th grade geometry.
Project Rubric (Formal)
3 parts fully modeled
Assembly fully constrained
Exploded drawing + BOM
Added to portfolio
Day 5
Assembly Assessment & Reflection
Assess
Students receive a set of 3 provided part files and must assemble them correctly in a timed assessment (one class period). They also produce an exploded view drawing. Reflection prompt in notebook: "How does thinking about assembly change the way you design individual parts?" This metacognitive question previews the concept of design intent that will deepen throughout the program.
Indiana Standards
CTE-TE 5.2 CTE-TE 6.1 IAS-ELA 8.W
IAS-ELA 8.W: Write explanatory texts to examine a topic ‚Äî the notebook reflection fulfills 8th grade informative writing while connecting to engineering metacognition.

Introduction to SolidWorks ‚Äî A Second CAD Language
Days 6‚Äì11 ¬∑ ~6 class sessions ¬∑ New Platform

Day 6
Why Two CAD Platforms? ‚Äî Industry Context & Interface Comparison
Intro
Teacher introduces why engineers need to be fluent in multiple CAD tools (Inventor is common in manufacturing/education; SolidWorks dominates product design, aerospace, and medical device industries). Students open SolidWorks for the first time and compare the interface to Inventor side-by-side: FeatureManager Design Tree vs. Browser Bar, CommandManager vs. Ribbon, Mate vs. Constraint terminology. Goal: leverage prior knowledge, don't start from zero.
Indiana Standards
CTE-TE 6.1 CTE-TE 2.1
CTE-TE 2.1: Influence of technology on history and society ‚Äî students discuss how SolidWorks shaped the product design industry and why platform fluency is a professional skill.
Comparison Activity
Students fill out a two-column "CAD Rosetta Stone" chart in their notebook mapping Inventor terms to their SolidWorks equivalents (sketch ‚Üí sketch, extrude ‚Üí boss-extrude, cut ‚Üí cut-extrude, etc.).
Day 7
SolidWorks Sketching ‚Äî Same Concepts, New Commands
Skill
Students recreate their Inventor Day 3 L-shape sketch in SolidWorks using Smart Dimension and sketch relations. Focus is on transfer of knowledge: students discover that the underlying concepts (fully defined sketch, geometric relations, parametric dimensions) are identical ‚Äî only the interface differs. Teacher emphasizes: "You already know how to think in CAD. You're just learning a new dialect."
Indiana Standards
CTE-TE 6.1 CTE-TE 5.1 IAS-Math 8.G
Assessment (Informal)
Sketch fully defined
Dimensions match spec
Day 8
Boss-Extrude, Cut-Extrude & Fillets in SolidWorks
Skill
Students work through a SolidWorks version of their Semester 2 part modeling exercises (extrude, cut, fillet) using a step-by-step tutorial sheet. Emphasis on the FeatureManager tree: understanding the order of features and how suppressing/unsuppressing a feature changes the model. Students model the same block-with-pocket they made in Inventor and compare the two files side by side.
Indiana Standards
CTE-TE 5.2 CTE-TE 6.1 IAS-Math 8.G
Assessment (Informal)
Features match Inventor version
FeatureManager organized
Day 9
SolidWorks Drawings & eDrawings Export
Skill
Students generate a SolidWorks Drawing (.slddrw) of their part, placing standard 3-view orthographic layout with dimensions. Introduction to eDrawings ‚Äî SolidWorks' lightweight viewer format used to share models with clients or manufacturers who don't have SolidWorks. Students export their drawing as a PDF and an eDrawing and add both to their portfolio, now showing the same design documented in two CAD platforms.
Indiana Standards
CTE-TE 5.1 CTE-TE 5.2 CTE-TE 4.2
Assessment (Informal)
3-view drawing correct
PDF + eDrawing exported
Day 10
SolidWorks Independent Design Challenge
Project
Students design an original part in SolidWorks with no tutorial guidance ‚Äî only a written prompt (e.g., "Design a small tool holder that mounts to the edge of a desk"). They must produce the part, a drawing, and a 3-sentence design rationale in their notebook. This is their first fully independent SolidWorks deliverable and goes directly into the portfolio.
Indiana Standards
CTE-TE 3.1 CTE-TE 5.2 CTE-TE 6.1

Design rationale written

SolidWorks Assessment ‚Äî Read a Drawing, Build the Part
Assess
Formal timed assessment: students receive a fully dimensioned orthographic drawing and must produce a correct SolidWorks model in one class period. Graded on dimensional accuracy, sketch constraints, feature organization, and file naming. This mirrors a real engineering "print reading" task and directly assesses both drawing interpretation and CAD execution.

Day 12
Why Code? ‚Äî Connecting Programming to Engineering
Intro
Teacher bridges from physical design to software: "Every product you'll make in this program will eventually need code ‚Äî to control motors, read sensors, or display data." Introduction to JavaScript as the first language: runs in any browser, immediate visual feedback, the language of the web and many microcontrollers. Students open browser DevTools console and type their first lines: alert(), console.log(), and basic math expressions. No setup required.
Indiana Standards
IAS-CS 8.AP.1 IAS-CS 8.AP.2
IAS-CS 8.AP.1: Create programs that include sequences, selections, and repetitions. IAS-CS 8.AP.2: Use variables and assignments to store and access data. These Computer Science standards anchor all JavaScript work in this and future semesters.
First Console Exercises
// Try these in the browser console:
console.log("Hello, Engineer!");
alert("My first popup");
let x = 5 + 3;
console.log(x);
Day 13
Variables, Data Types & Operators
Skill
Students learn let, const, and var declarations; string, number, and boolean data types; arithmetic and comparison operators. They complete a guided worksheet: declare variables for a "product spec" (name, width, height, weight, inStock) and write expressions that calculate area and check if the product fits in a box. Engineering context is deliberate ‚Äî variables mirror the parametric dimensions they already use in CAD.
Indiana Standards
IAS-CS 8.AP.2 IAS-Math 8.EE
IAS-Math 8.EE (Expressions & Equations): Work with radicals and integer exponents, linear equations. Variables in JavaScript directly mirror algebraic variables ‚Äî students see the same logic they use in math class applied in code.
Assessment (Informal)
Variables correctly declared
Area calculation outputs correctly
Day 14
Functions ‚Äî Reusable Instructions
Skill
Students learn to write and call functions with parameters and return values. Engineering framing: "A function is like a reusable machine ‚Äî you feed it inputs, it produces outputs." Exercises include: a function that calculates the volume of a rectangular prism (given length, width, height), a function that checks if a dimension is within tolerance, and a function that returns a material recommendation based on weight. Students document each function with a comment block above it.
Indiana Standards
IAS-CS 8.AP.1 IAS-CS 8.AP.2 IAS-Math 8.F
IAS-Math 8.F (Functions): Define, evaluate, and compare functions. A JavaScript function is a direct computational analog of a mathematical function ‚Äî same input‚Üíoutput logic. This cross-curricular connection is explicitly discussed.
Code Snippet Preview
// Volume calculator function
function calcVolume(l, w, h) {
  return l * w * h;
}
console.log(calcVolume(4, 3, 2));
Day 15
Conditionals & Loops ‚Äî Making Decisions & Repeating Tasks
Skill
Students learn if/else statements and for loops. Engineering contexts: an if/else that classifies a material as "lightweight," "medium," or "heavy" based on density input; a for loop that prints a multiplication table for drill bit sizes. Students also encounter their first logic bug ‚Äî teacher intentionally introduces a broken loop and students practice debugging using console.log() to trace values. Debugging is framed as iteration ‚Äî the same mindset as the EDP.
Indiana Standards
IAS-CS 8.AP.1 IAS-CS 8.AP.3 IAS-Math 8.EE
IAS-CS 8.AP.3: Identify and fix errors in algorithms and programs. Debugging directly fulfills this standard ‚Äî students learn systematic error-tracing using console output.
Assessment (Informal)
Conditional logic correct
Loop runs correct iterations
Bug identified & fixed
Day 16
DOM Manipulation ‚Äî Making a Webpage React to Input
Skill
Students write their first interactive HTML+JS page: a simple "Part Spec Calculator" where a user enters length, width, and height into input boxes and clicks a button to calculate and display volume, surface area, and a weight estimate (given a density). Students use document.getElementById(), addEventListener(), and innerHTML. This is their first full input‚Üíprocess‚Üíoutput program ‚Äî the core pattern for all future sensor/display projects.
Indiana Standards
IAS-CS 8.AP.1 IAS-CS 8.AP.2 CTE-TE 6.1
The DOM calculator is a digital tool that models an engineering calculation (CTE-TE 6.1), built with sequences, functions, and events (IAS-CS 8.AP.1 & 8.AP.2).
Assessment (Informal)
Button triggers calculation
Output displays on page
Math is correct
Day 17
JavaScript Formal Assessment ‚Äî Build a Mini-Calculator
Assess
Formal assessment: students independently build a working JavaScript calculator page that takes two dimensions as input and outputs: (1) their product, (2) a pass/fail check against a max value, and (3) a text recommendation. Graded on: correct logic, working DOM interactions, clear variable naming, and at least one function definition. Students submit the .html file. No AI tools permitted on this assessment ‚Äî establishes baseline coding ability before AI-assisted coding is introduced in Semester 4.
Indiana Standards
IAS-CS 8.AP.1 IAS-CS 8.AP.2 IAS-CS 8.AP.3 IAS-Math 8.EE

Day 18
Integration Challenge ‚Äî Design a Part & Write Its Spec Sheet in Code
Project
Students design a small original part in SolidWorks, then build a companion JavaScript webpage that acts as an interactive "spec sheet" for that part: user inputs a desired scale factor and the page recalculates all dimensions, volume, and estimated weight dynamically. The pairing of physical CAD model + interactive digital spec tool previews the kind of integrated product documentation used in real engineering firms.
Indiana Standards
CTE-TE 3.1 CTE-TE 5.2 CTE-TE 6.1 IAS-CS 8.AP.1 IAS-CS 8.AP.2 IAS-Math 8.EE
This project is the first true multi-domain deliverable: EDP thinking, CAD modeling, technical documentation, and programming all converge. It directly previews the Sophomore capstone structure.
Project Rubric (Formal)
SolidWorks part complete
JS spec sheet functional
Scale calculation correct
Added to portfolio
